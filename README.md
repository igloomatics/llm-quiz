# 大模型八股面经（LLM Interview Playbook）

一个面向大模型学习者的互动刷题网站：覆盖 Transformer、BERT、多模态、强化学习与人类对齐等核心知识，用「做题 + 解析 + 错题复习」帮助你高效备战面试与系统化复习。

在线体验：<https://igloomatics.github.io/llm-quiz/>

---

## 创立动机

我希望把这个项目建设成一个 **LeetCode 风格的大模型学习与面试平台**：

- 不只是“看八股”，而是通过持续刷题形成结构化能力
- 用题单、进度、错题与复盘，把零散知识变成可量化成长
- 让学习者像刷算法题一样，系统拿下 LLM 面试核心知识

## 项目简介

`llm-quiz` 是一个纯前端静态项目，不依赖后端服务，开箱即用，天然适配 GitHub Pages。

题库目前包含：

- **166 道题**
- **17 个专题**（如 Transformer 入门/进阶、手撕 Transformer、BERT、ViT、CLIP、BLIP、强化学习、人类对齐等）
- **8 种题型**（选择、多选、代码填空、代码默写、填空、简答、提示、简介）

---

## 功能亮点

- **多维筛选**：按专题和难度快速过滤题目
- **错题复习模式**：自动汇总错题，答对即移出错题本
- **代码题支持**：代码填空、代码默写、答案对照
- **专题导读页**：做题前先看核心概念与背景
- **题目顺序控制**：支持随机打乱与恢复顺序
- **学习资源聚合**：论文、代码仓库、教程一键直达

## 本地运行

这是纯静态网站，直接打开即可运行：

1. 克隆仓库
2. 双击打开 `index.html`（或使用 Live Server）
3. 开始刷题

如需维护题库：

1. 打开 `question-manager.html`
2. 导入当前 `questions.js`
3. 在可视化界面编辑/新增/排序题目
4. 导出新的 `questions.js` 覆盖原文件

---

## 部署到 GitHub Pages

> 适用于仓库名为 `llm-quiz` 的项目页部署。

1. 将代码推送到 GitHub 仓库（例如：`https://github.com/<你的用户名>/llm-quiz`）
2. 进入仓库 `Settings` → `Pages`
3. 在 `Build and deployment` 中选择：
   - `Source`: `Deploy from a branch`
   - `Branch`: `main`（或你的默认分支）
   - `Folder`: `/ (root)`
4. 保存后等待 1-3 分钟，访问：
   - `https://<你的用户名>.github.io/llm-quiz/`

如果你使用自定义域名，可在 `Pages` 中配置 `Custom domain`。

## 建设建议（向 LeetCode 风格演进）

### 产品层（先做最有杠杆的功能）

- **题单系统**：按岗位/阶段设计题单（算法岗、应用岗、Agent/RAG专项）
- **用户进度**：已做、已掌握、重做次数、连续打卡天数
- **复盘闭环**：错题本 + 收藏夹 + “7天后重练”机制
- **评测标准化**：每题明确考点、难度、高频追问、面试官视角

### 技术层（建议的迭代顺序）

- **第 1 阶段（当前）**：静态站点 + 本地题库（已具备）
- **第 2 阶段**：引入后端与账号（登录、进度同步、云端错题）
- **第 3 阶段**：在线判题/代码运行环境（代码题自动评测）
- **第 4 阶段**：社区能力（题解、讨论、贡献题目审核）

### 协作与维护

- 想增加专题：直接在 `questions.js` 追加 `topic` 即可自动出现在筛选里
- 想做团队共建：建议用 PR 流程维护题库，避免直接覆盖冲突
- 建议新增 `CONTRIBUTING.md`：统一题目格式、命名规范和审核流程
